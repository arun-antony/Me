Google Cloud Basics

Summary of Google cloud basics. Summary authored by Arun Antony

On Prem Systems Challenges (Before cloud)

  1. High cost of infrastructure
  2. Needs ahead of time planning for future load increase for example
  3. Low infrastructure utilization during off peak time
  4. Dedicated infrastructure maintenence team required

How Cloud overcomes the challenges of on-prem systems

  1. Cloud Providers maintains and provisions the servers to clients who need them on demand
  2. Massively ecconomical due to scale
  3. No need to plan for future as Scaling is done by cloud systems
  4. No need to spend money for maintainence
  5. Go global in minutes

Compute Engine Fundamentals

  1. In on prem systems, applications are deployed to physical servers
  2. In cloud, clients can rent virtual servers on GCP - Virtual Machines -
  3. Google Compute Engine - Provision & Manage Virtual machines
Compute Engine Features
  1. Create and manage lifecycle of VM instances
  2. Load Balancing and Auto Scaling for multiple VM instances
  3. Attach storage & network storage to your VMs
  4. Manage n/w connectivity and configuration of VMs
  5. SSH to connect to the VM instances

Google CLoud CLI

The general syntax is as follows

gcloud group subgroup command options

Group: config, compute

Subgroup: 

        gcloud config list  // This command lists the current activated configuration 
        gcloud config list project // This list the project name mentioned in the active configuration
        gcloud config configurations list // this lists all configurations 
        gcloud config configurations activate [configuration-name] //This activates the configuration that has the provided name
        gcloud config configurations describe [configuration-name] //This provides details of the configiration that has the provided Name

        
        gcloud compute instances list //Lists all the VM instances
        gcloud compute instances create //Creates a VM with default params, region and zone will be picked from the configuration
        gcloud compute instances create [NAME] //Creates a VM instance with providd name and other default params
        gcloud compute instances describe [NAME] //Prints details of the VM with given name


Setting UP

  1. Create GCP account - google account
  2. Regions and Zones - Helps tackle issues of High Latency (response time) and Availability
  3. Regions
    • High Availability
    • Low Latency
    • Global Foorprint
    • Adhere to govt. regulations
  4. Zones
    • High Availability in the same region or geographical location
    • Increased availability and fault tolerance within same region
    • Each zone has one or more discrete cluster - cluster is a distinct physical infrstructure that is housed in a data center
    • Zones in region are connected through low-latency links
  5. Check for Free Tier setups
  6. Billing and free credits and usage monitoring
