Google Cloud Basics

Summary of Google cloud basics. Summary authored by Arun Antony

On Prem Systems Challenges (Before cloud)

  1. High cost of infrastructure
  2. Needs ahead of time planning for future load increase for example
  3. Low infrastructure utilization during off peak time
  4. Dedicated infrastructure maintenence team required

How Cloud overcomes the challenges of on-prem systems

  1. Cloud Providers maintains and provisions the servers to clients who need them on demand
  2. Massively ecconomical due to scale
  3. No need to plan for future as Scaling is done by cloud systems
  4. No need to spend money for maintainence
  5. Go global in minutes

Setting UP

  1. Create GCP account - google account
  2. Regions and Zones - Helps tackle issues of High Latency (response time) and Availability
  3. Regions
    • High Availability
    • Low Latency
    • Global Foorprint
    • Adhere to govt. regulations
  4. Zones
    • High Availability in the same region or geographical location
    • Increased availability and fault tolerance within same region
    • Each zone has one or more discrete cluster - cluster is a distinct physical infrstructure that is housed in a data center
    • Zones in region are connected through low-latency links
  5. Check for Free Tier setups
  6. Billing and free credits and usage monitoring
