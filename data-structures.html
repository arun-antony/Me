Main points

  • Intro to DS
    1. What is DS? It can be defined as arrangement of data in memory so that a program could execute the code on some data most efficiently. A program needs data for it to work. So without DS, a program cannot exist.
    2. Memory is allocated to a program for its execution. When a program is started, it is allocated 3 sections. Code section, stack and heap.
    3. Code Section: Program executable or machine level code is brought to code section of RAM when a program is started.
    4. Stack: Static allocated memory. For each function that is started are allocated memory in stack - called as Stackframe or Activation record. So the memory known at compile time is allocated here. This area can be directly accessed by the code. Primary variables, pointers etc are allocated here. References to Dynamicaly allocated memory locations and resources are present here.
    5. Heap: Dynamically allocated memory: Arrays, Linked list etc are allocated and deallocated on demand.
    6. Monolithic programming: All code written in a single function. Not reusable and inneficient
    7. Procedural or Modular programming: Split into multiple functions, classes - modular, reusable, maintainable
    8. Interesting Fact in C++: Code that uses references become inline - no separate memory is allocated in stack - it becomes part of code section.
    9. Call by value: Copies of variable are passed.
      Call by reference: References are passed. So the function works on the same copy of variables. Such fns become inline. No separate activation records allocated.
      Call by address: Pointers are passed. A pointer memory is allocated for the function in its Activation record.
    10. Physical vs Logical Data Structures:
      Physical DS: They define how the memory is organized: Arrays (if size id known), Linked List (if size is not known upfront).
      Logical DS: They are DS derived or created using physical DS. Eg:
      1. Stack - LIFO , Queue - FIFO - Linear DS
      2. Graph, Tree - Non Linear DS
      3. HashTable - Linear/Non-Linear/Tabular DS

      ADT - Abstract Datatypes: Definition of data, Operations on Data : Impl details are abstract eg: Class

  • Complexities
    1. Time Complexity: Representation of time required for exectution of a piece of code. It is represented as (ndeg)
    2. Space Complexity: Representaton of space or memory used for execution of a piece of code.
